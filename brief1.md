# Brief: Telegram Mini App «Диагностика бизнеса»

**Проект:** Квиз «Насколько ваш бизнес зависит от вас»
**Формат:** Telegram Mini App (TMA)
**Стек:** Vanilla JS + CSS, без фреймворков
**Бандл:** < 300 KB
**Язык:** Русский

---

## Архитектура приложения

```
Экран 0: Приветствие
    ↓ [MainButton «Начать диагностику»]
Экран 1–6: Вопросы (автопереход 500–600 мс)
    ↓ [после ответа на вопрос 6]
Экран 7: Загрузка результата (2–3 сек)
    ↓ [автоматически]
Экран 8: Результат
    ↓ [MainButton «Разобрать мою ситуацию»]
Экран 9: CTA / Запись
    + [параллельно] Бот отправляет карточку в чат
```

## Технические требования

### Нативные элементы Telegram (обязательно)

```js
const tg = window.Telegram.WebApp;
tg.ready();                          // сразу после загрузки
tg.expand();                         // развернуть на весь экран

// Тема
tg.themeParams.bg_color              // фон
tg.themeParams.text_color            // основной текст
tg.themeParams.hint_color            // серый текст
tg.themeParams.button_color          // цвет акцента
tg.themeParams.secondary_bg_color    // фон карточек

tg.onEvent('themeChanged', applyTheme);  // реакция на смену темы

// MainButton — менять текст по контексту
tg.MainButton.setText('Начать диагностику');
tg.MainButton.show();
tg.MainButton.showProgress();        // спиннер на экране загрузки

// BackButton
tg.BackButton.show();
tg.BackButton.hide();
tg.BackButton.onClick(goBack);

// Haptic
tg.HapticFeedback.impactOccurred('light');          // при ответе
tg.HapticFeedback.notificationOccurred('error');    // красная зона
tg.HapticFeedback.notificationOccurred('warning');  // жёлтая
tg.HapticFeedback.notificationOccurred('success');  // зелёная

// Данные пользователя
const user = tg.initDataUnsafe.user;
const firstName = user?.first_name ?? '';

// Сохранение прогресса
tg.CloudStorage.setItem('quiz_progress', JSON.stringify({step, answers}));
tg.CloudStorage.getItem('quiz_progress', (err, val) => { ... });
```

### Бэкенд (минимальный)

```
POST /api/result
  Body: { initData, answers[], score, level }
  → верифицировать initData через HMAC-SHA256
  → сохранить результат в БД
  → отправить карточку через Telegram Bot API
  → вернуть { ok: true }
```

### UTM-трекинг

```
Ссылка с лендинга: t.me/botname?start=from_landing
Ссылка из соцсетей: t.me/botname?start=from_instagram
```
`startParam` передаётся в `initDataUnsafe.start_param` — сохранять в БД вместе с результатом.

### Аналитика (Яндекс.Метрика)

Событие на каждый шаг:
```js
ym(COUNTER_ID, 'reachGoal', 'quiz_step_1');
ym(COUNTER_ID, 'reachGoal', 'quiz_complete');
ym(COUNTER_ID, 'reachGoal', 'cta_click');
```

## Чего НЕ будет в первой версии

| Что | Почему нет |
|---|---|
| Регистрация и личный кабинет | Лишняя сложность, Telegram даёт ID автоматически |
| Ветвление вопросов | Все 6 вопросов одинаковые для всех, проще и быстрее |
| Платный расширенный отчёт | MVP — генерация лидов, не монетизация |
| Анимации переходов между экранами | V2, не влияет на конверсию критично |
| Голосовые команды | TMA не поддерживает напрямую |
| A/B тест внутри приложения | V2, сначала нужна аналитика |
| Второй язык (English) | Аудитория — рунет |
| PDF-отчёт | Бот-карточка в чате достаточна для MVP |
| Сбор email | Telegram даёт контакт автоматически |
| Вебинарная воронка | Отдельный продукт, не для этого квиза |

## Порядок разработки

### Этап 1 — Статика (без бэкенда)
1. `index.html` с базовой структурой
2. `styles.css` с CSS-переменными из themeParams (тёмная/светлая тема)
3. Экран 0: приветствие, MainButton
4. Экраны 1–6: вопросы с прогресс-баром и автопереходом
5. Экран 7: загрузка с анимацией
6. Экран 8: результат с тремя уровнями (пока без имени)
7. Экран 9: CTA с openTelegramLink

### Этап 2 — Нативность
8. Подключить `tg.initDataUnsafe.user` → имя на результате
9. HapticFeedback на каждый ответ и результат
10. BackButton на вопросах 2–6
11. CloudStorage: сохранение прогресса (если закрыл и переоткрыл)
12. themeChanged: реакция на смену темы

### Этап 3 — Бэкенд
13. POST /api/result с верификацией initData
14. Telegram Bot API: отправка карточки в чат
15. Сохранение результата + startParam в БД
