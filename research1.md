# Исследование: Telegram Mini Apps в нише AI-автоматизации и бизнес-консалтинга

---

## Контекст проекта

**Квиз «Диагностика: насколько бизнес зависит от вас»** — AI-квиз как Telegram Mini App.
5–7 вопросов → персональный мини-отчёт → CTA на консультацию.
Продолжение лендинга: секция «Боли» → квиз конкретизирует боль → предложение решения.

---

## 5 примеров Telegram Mini Apps

---

### 1. Syntx AI (@syntxaibot)

**Что делает:**
Агрегатор из 90+ AI-инструментов прямо в Telegram. Единый доступ к ChatGPT, Claude 3.5, Gemini, Midjourney, DALL-E, Runway и другим моделям без VPN и отдельных регистраций. 213 000 пользователей.

**Сильные стороны:**
- Одна точка входа для всех моделей — устраняет «зоопарк подписок»
- Нет блокировок VPN — критично для российской аудитории
- Удобная категоризация инструментов
- Реферальная программа снижает CAC

**Слабые стороны:**
- Бесплатно только 5 токенов — кончаются мгновенно
- Запутанная система тарифов (5 уровней от €9 до €119)
- Нет бизнес-функций: нет воронок, CRM, сегментации
- Ориентирован на контент-мейкеров, не на руководителей бизнеса

**Что сделать лучше:**
Бизнес-шаблоны (КП, анализ конкурентов, разбор договоров), три чётких тарифа вместо пяти, командный аккаунт для малого бизнеса.

---

### 2. GigaChat (@gigachat_bot) — Telegram Mini App от Сбера

**Что делает:**
Официальное мини-приложение нейросети GigaChat. Генерация текста и изображений, анализ документов, голосовые команды. Полностью русскоязычная модель.

**Сильные стороны:**
- Единственная крупная русскоязычная модель в TMA — понимает юридические и бизнес-термины РФ
- Анализ документов: загрузил договор → краткое содержание и риски
- Голосовые команды удобны «на ходу»
- Бесплатный базовый доступ, нет проблем с санкциями

**Слабые стороны:**
- Качество аналитики уступает GPT-4 / Claude
- Ограниченный контекст разговора
- Нет интеграций с внешними сервисами
- Интерфейс аскетичен: нет шаблонов, истории, управления сессиями

**Что сделать лучше:**
Готовые бизнес-промпты (оферта, анализ NPS, ТЗ), история и организация диалогов, интеграция с «Сбер Бизнес».

---

### 3. Marquiz в Telegram — квиз-воронки для лидогенерации

**Что делает:**
Конструктор маркетинговых квизов с интеграцией в Telegram. Квизы открываются прямо в мессенджере через бота. Telegram автоматически подставляет логин — не нужно вводить телефон. Средняя конверсия ~40% из просмотра в лид.

**Сильные стороны:**
- Квиз не выводит пользователя из Telegram — конверсия выше
- Автозаполнение контактных данных снижает отказы
- Готовые шаблоны для разных ниш
- Интеграция с CRM, Google Sheets — лиды попадают прямо в базу
- Визуальный редактор без кода

**Слабые стороны:**
- Не нативный TMA, а встроенная ссылка через бота — ограниченная интеграция
- Нет AI-персонализации: вопросы статичны, ветвление базовое
- Для сложных диагностических квизов функционала недостаточно
- Нет автоматической нарративной интерпретации результатов — логику надо писать вручную

**Что сделать лучше:**
AI-генерация персонализированного результата («Ваш тип — X, вот три шага»), углублённое ветвление, нативный TMA-режим, интеграция с Telegram Stars для платных диагностик.

---

### 4. Tribute (@subscribeappbot) — монетизация для экспертов и консультантов

**Что делает:**
Telegram-нативный инструмент монетизации. Продажа подписок на закрытые каналы, донаты, цифровые продукты, кастомные заказы. Выплатил >$45 млн более чем 70 000 авторам. Комиссия 10%.

**Сильные стороны:**
- Максимально нативный: всё внутри Telegram, аудитория не уходит
- Быстрая настройка (несколько минут)
- Мультивалютность: карты любого банка + крипта (USDT/BTC/TON)
- Автоматическое управление доступом: заплатил → добавлен; подписка кончилась → удалён
- Прозрачная аналитика доходов

**Слабые стороны:**
- Минимальный порог выплаты €100 — проблема для начинающих
- Максимальная цена подписки €3 000 — ограничивает premium B2B-консалтинг
- Нет CRM и управления лидами — только уже купившие клиенты
- Нет инструментов диагностики и квизов — нельзя привлекать новых, только монетизировать существующих

**Что сделать лучше:**
Мини-лендинг внутри Telegram для прогрева до покупки, квиз-онбординг для новых подписчиков, интеграция с квиз-платформами для квалификации лидов перед продажей.

---

### 5. PuzzleBot — конструктор TMA-воронок для бизнеса (puzzlebot.top)

**Что делает:**
No-code платформа для создания Telegram-ботов и мини-приложений. 67 000 разработчиков. Полные воронки продаж: квизы, магазины, запись на услуги, рассылки, приём платежей (ЮKassa, Stripe, крипта), интеграция с 2000+ сервисами через Zapier/Make.

**Сильные стороны:**
- Полный стек «бот + мини-приложение + воронка» в одном инструменте
- Квизы вместо форм повышают конверсию регистраций на 10–15%
- Цена ~891 ₽/мес — доступно для малого бизнеса
- Готовые шаблоны для разных ниш
- Активное сообщество, регулярные обновления

**Слабые стороны:**
- Сложная логика ветвления — нужны технические знания
- Нет глубокой AI-интеграции из коробки
- Аналитика базовая, нет бизнес-метрик
- Нет встроенного A/B-тестирования воронок

**Что сделать лучше:**
AI-конструктор квизов (описал нишу → получил готовый квиз), аналитика воронки с визуализацией этапов, шаблон «диагностика → результат → продажа консультации» под B2B и консалтинг.

## Типичные экраны в подобных приложениях

### Обязательные экраны:
1. **Приветственный экран** — заголовок, 1–2 строки о чём квиз, кнопка «Начать»
2. **Экран вопроса** — вопрос + варианты ответа (кнопки или шкала), прогресс-бар (шаг X из Y)
3. **Переходный экран** (опционально) — «Отлично, ещё пара вопросов…» — снижает отток
4. **Экран загрузки результата** — «Анализируем ваши ответы…» — создаёт ощущение AI-обработки
5. **Экран результата** — уровень/тип + короткое описание + 2–3 персональные рекомендации
6. **CTA-экран** — оффер на консультацию + кнопка записи / контакт

### Обязательные элементы:
- **Прогресс-бар** — снижает тревогу («сколько ещё?»), повышает дохождение
- **Один вопрос на экране** — не перегружать, мобильный формат
- **Крупные кнопки ответов** — не мелкие чекбоксы, а tapable-кнопки на весь экран
- **Персонализированный заголовок результата** — «Ваш уровень: Критическая зависимость» лучше, чем «Результат теста»
- **Визуальный индикатор уровня** — шкала, диаграмма или цвет (красный/жёлтый/зелёный)
- **Конкретные рекомендации** (не общие слова) — «Вам стоит начать с делегирования этих 3 функций»
- **CTA с низким барьером** — «Получить разбор на бесплатной консультации» > «Купить»

### Что отличает хорошие TMA от плохих:
- Быстрая загрузка (< 2 сек) — иначе закрывают
- Нативный стиль Telegram (тёмная/светлая тема, шрифты) — не «чужеродный» сайт внутри мессенджера
- Минимум полей для заполнения — Telegram подставляет данные автоматически
- Анимации переходов — ощущение живого приложения, не статичной страницы
- Мгновенная обратная связь на каждый ответ — выбранный вариант подсвечивается

## Ключевые выводы по конкурентам

1. **Разрыв в рынке**: нет ни одного TMA, который сочетает AI-диагностику + квиз-воронку + продажу консультации в одном нативном приложении для B2B-аудитории.

2. **Монетизация зрелая, лидогенерация слабая**: Tribute хорошо продаёт, но не привлекает. Marquiz хорошо привлекает, но не монетизирует. Связки нет — это наша ниша.

3. **AI как дифференциатор**: приложения с AI-персонализацией результатов («ваш тип бизнеса — X, ваш главный риск — Y») конвертируют значительно лучше статичных форм. Это массово не реализовано.

4. **Ниша консалтинга не закрыта**: большинство TMA ориентированы на крипту, игры и e-commerce. Серьёзных продуктов для B2B-консультантов практически нет.

5. **Нативность = конверсия**: приложения, которые не выводят пользователя из Telegram, конвертируют на 30–40% лучше редиректов на внешние сайты.

## Структура нашего квиза: конкретный план

### Концепция

**Название:** «Диагностика: насколько бизнес зависит от вас»
**Формат:** 6 вопросов → балльная система → 3 уровня результата → персональный мини-отчёт → CTA на консультацию
**Тон:** Серьёзный, но не пугающий. Диагностический, не осуждающий. Как хороший врач — говорит правду, но с уважением.

---

### Вопросы и варианты ответов

Каждый ответ приносит 0–3 балла. Максимум — 18 баллов.

---

_Сколько решений в вашей компании требуют вашего личного участия?_

- Почти все — без меня ничего не движется **[3 балла]**

- Только стратегические вопросы **[1 балл]**
- Команда действует самостоятельно **[0 баллов]**

---

- Несколько вещей решаются без меня, но не всё **[1 балл]**
- Компания работает как часы **[0 баллов]**

---

- Полная база знаний, регламенты, скрипты **[0 баллов]**

---

- Несколько раз в неделю **[2 балла]**
- Редко, только в нестандартных ситуациях **[1 балл]**

- Сильно потеряет в стоимости без меня **[2 балла]**
- Некоторое время проработает самостоятельно **[1 балл]**
- Работает как актив, независимо от меня **[0 баллов]**

### Три уровня результата

#### Красная зона: 13–18 баллов
**«Вы — главный актив и главный риск»**

Бизнес держится на вас. Вы принимаете большинство решений, процессы живут у вас в голове, без вас работа останавливается. Это не просто дискомфорт — это ограничение роста. Масштабировать такой бизнес сложно, продать — почти невозможно.

_Рекомендации:_
1. Начать с аудита: какие решения вы принимаете каждую неделю и можно ли их делегировать
2. Выбрать 1–2 повторяющихся процесса и задокументировать их — это первый шаг к делегированию
3. Посмотреть, где AI может взять на себя рутину прямо сейчас (ответы на типовые вопросы, сортировка запросов, первичный анализ данных)

#### Жёлтая зона: 7–12 баллов
**«Зависимость есть — но её можно снизить за 2–3 месяца»**

Бизнес частично работает без вас, но вы всё ещё перегружены операционкой. Есть конкретные точки, где вы — узкое горлышко. Хорошая новость: у вас уже есть база для изменений.

_Рекомендации:_
1. Найти 2–3 «горлышка» — где команда чаще всего ждёт именно вас
2. Внедрить базовые AI-инструменты для делегирования рутины
3. Выстроить систему «решения по умолчанию» — чтобы команда знала, как действовать без вас в типовых ситуациях

#### Зелёная зона: 0–6 баллов
**«Хороший задел — следующий шаг: масштаб»**

У вас выстроена база. Команда работает самостоятельно, процессы задокументированы. Следующий вопрос не «как освободиться», а «как вырасти без пропорционального роста затрат и команды». Здесь AI-автоматизация даёт максимальный эффект.

_Рекомендации:_
1. Провести аудит процессов на предмет автоматизации — что можно убрать из зоны ответственности людей совсем
2. Внедрить AI-агентов для обработки входящих запросов, отчётности, аналитики
3. Сфокусировать своё время только на стратегии и точках роста

## Детальный разбор экранов

### Экран 0: Приветствие

**Задача:** Зацепить, объяснить ценность, снизить барьер входа.

```
[Иконка или иллюстрация — бизнесмен / весы / здание]

Диагностика:
насколько ваш бизнес
зависит от вас?

6 вопросов → персональный отчёт
о рисках и точках роста

[Кнопка: Пройти диагностику →]

_Займёт 2 минуты_
```

**Важно:** Кнопка — MainButton Telegram (синяя, фиксированная внизу). Не кастомная — нативная. Это доверие.

---

### Экраны 1–6: Вопросы

**Задача:** Минимальное трение, один выбор за раз.

```
[Прогресс-бар: ██████░░░░░░  Шаг 2 из 6]

Что происходит, если вы
уезжаете в отпуск на две недели?

[ Работа фактически останавливается ]
[ Мне постоянно звонят и пишут     ]
[ Некоторые вопросы решают без меня ]
[ Компания работает как часы        ]
```

**Правила экрана вопроса:**
- Прогресс-бар обязателен — люди бросают, если не видят конца
- Кнопки ответов на всю ширину экрана — tap target крупный
- При выборе ответ подсвечивается (цвет акцента) — немедленный фидбек
- Автопереход к следующему вопросу через 300–500 мс после выбора (не нужна кнопка «Далее»)
- BackButton Telegram работает — можно вернуться на предыдущий вопрос

---

### Экран 7: Загрузка результата (обязательный!)

**Задача:** Создать ощущение реальной обработки. Это не анимация ради анимации — это психологический триггер.

```
[Анимированный индикатор — пульсирующий круг или прогресс]

Анализируем ваши ответы...

✓ Оцениваем уровень зависимости
✓ Определяем ключевые риски
⟳ Формируем рекомендации...

[Длительность: 2–3 секунды]
```

**Почему важно:** Мгновенный результат воспринимается как «просто программа». Небольшая пауза с прогрессом — как «настоящий анализ». Конверсия в прочтение результата выше.

---

### Экран 8: Результат

**Задача:** Персонализированный удар в боль + надежда + конкретика.

```
[Цветовой индикатор: красный/жёлтый/зелёный круг]

Ваш результат: 14 из 18

🔴 Критическая зависимость

«Вы — главный актив
и главный риск бизнеса»

━━━━━━━━━━━━━━━━━━━━━

Что это значит:
Без вас работа останавливается.
Масштабировать и продать такой
бизнес практически невозможно.

━━━━━━━━━━━━━━━━━━━━━

Три первых шага:
→ Провести аудит решений
→ Задокументировать 1–2 процесса
→ Внедрить AI для рутины

[Кнопка: Разобраться на консультации →]
```

**Правила экрана результата:**
- Цифра балла + визуальный индикатор уровня (цвет, шкала)
- Персонализированный заголовок — не «Результат», а «Вы — главный актив и главный риск»
- Максимум 3 рекомендации — не 10. Конкретные, не абстрактные
- Одна кнопка CTA — не два варианта

---

### Экран 9: CTA (может быть частью экрана результата)

**Задача:** Снять страх перед следующим шагом. Бесплатно + конкретно + быстро.

```
━━━━━━━━━━━━━━━━━━━━━

Хотите конкретный план
для вашей ситуации?

На бесплатной консультации (30 мин)
разберём ваши точки зависимости
и наметим первые шаги автоматизации.

[Кнопка: Записаться на консультацию →]

или

[Написать Наталье напрямую →]
```

**Варианты CTA по уровню барьера (от низкого к высокому):**
1. «Написать в Telegram» — самый низкий барьер
2. «Записаться на бесплатную консультацию» — средний
3. «Получить разбор по вашей ситуации» — формулировка снижает барьер

## Конверсионные бенчмарки и психология квизов

### Цифры, на которые ориентироваться

| Метрика | Обычная форма | Quiz-воронка | TMA-квиз |
|---|---|---|---|
| Completion rate | — | 60–80% | ~75–85% (нативный UX) |
| Трафик → лид | 1–3% | 15–30% | выше за счёт нативности |
| Завершил → оставил контакт | — | 30–50% | авто — Telegram даёт данные |

Коучи и консультанты с assessment-квизами получают в **3–5x больше лидов** по сравнению с обычными формами (данные ScoreApp, Interact, LeadQuizzes).

### Что повышает completion rate

- **Прогресс-бар** — рост завершаемости на **20–28%** (эффект Зейгарник: незавершённое действие вызывает психологический дискомфорт)
- **Первый вопрос лёгкий** — каждый данный ответ усиливает желание дойти до конца (принцип consistency, Чалдини)
- **Автопереход** без кнопки «Далее» — убирает трение, ускоряет ритм
- **Curiosity gap** в заголовке: «Узнайте, почему бизнес не растёт» > «Пройдите тест»
- **Социальное доказательство** в интро: «2 400 предпринимателей уже прошли» снижает барьер

### Что делает экран результата эффективным

- **Называть тип, не цифру**: «Критическая зависимость» работает лучше, чем «14 из 18 баллов» — запоминается и хочется переслать
- **Зеркало проблемы**: 2–3 предложения, которые описывают ситуацию так точно, что пользователь думает «это про меня» — создаёт доверие
- **Разбивка по зонам** (что сильно / что провисает) лучше единого вывода — конкретнее
- **Визуальный индикатор** (шкала, цвет) — показывает разрыв между «сейчас» и «идеально», формирует мотивацию

### CTA: что конвертирует лучше всего

| Вариант CTA | Конверсия в заявку |
|---|---|
| Бесплатная консультация 15–20 мин | **25–40%** — лучший результат для услуг |
| PDF-отчёт / чеклист под профиль | 40–60% в email, но ниже в оплату |
| Кнопка в Telegram (бот, диалог) | 50–70% клик-рейт внутри TMA |
| Вебинар / мастер-класс | 10–20%, нужен отдельный прогрев |

**Вывод для нашего квиза:** оптимальный CTA — «Разобрать вашу ситуацию на бесплатном звонке 20 минут» с прямой кнопкой в Mini App. Добавление дефицита («2 места на этой неделе») даёт ещё +15–25% к конверсии.

## Технические требования и особенности TMA

### Что Telegram даёт автоматически (ничего не спрашивать у пользователя)

Из `window.Telegram.WebApp.initDataUnsafe.user`:
- `id` — уникальный Telegram ID (использовать как идентификатор, не username)
- `first_name`, `last_name` — имя для персонализации («Ваш результат, Анна:»)
- `language_code` — язык (`ru`, `en`) — автовыбор языка квиза
- `username` — @ник (может отсутствовать — не надёжен как идентификатор)
- `is_premium` — есть ли Telegram Premium

**Важно:** данные подписаны HMAC-SHA256 и **обязательно верифицируются на сервере** через бот-токен. Нельзя доверять `initDataUnsafe` без проверки.

### MainButton и BackButton

```js
const tg = window.Telegram.WebApp;

// Большая нативная кнопка внизу — менять текст по контексту
tg.MainButton.setText('Начать диагностику');
tg.MainButton.show();
tg.MainButton.onClick(() => { startQuiz(); });
tg.MainButton.showProgress(); // спиннер пока «считаем результат»

// Стрелка назад в шапке — показывать на вопросах 2+
tg.BackButton.show();
tg.BackButton.onClick(() => { goToPrevQuestion(); });
tg.BackButton.hide(); // скрыть на старте и на результате
```

Логика по экранам:
| Экран | MainButton | BackButton |
|---|---|---|
| Приветствие | «Начать диагностику» | скрыта |
| Вопрос 1 | скрыта (автопереход) | скрыта |
| Вопросы 2–6 | скрыта (автопереход) | показана |
| Загрузка | showProgress() | скрыта |
| Результат | «Записаться на консультацию» | скрыта |

### Тема (dark/light) — автоматически

```js
// Использовать CSS-переменные из themeParams
document.body.style.setProperty('--bg', tg.themeParams.bg_color);
document.body.style.setProperty('--text', tg.themeParams.text_color);
document.body.style.setProperty('--button', tg.themeParams.button_color);

// Реакция на смену темы
tg.onEvent('themeChanged', () => applyTheme());
```

### Haptic Feedback — делает интерфейс живым

```js
// При выборе ответа
tg.HapticFeedback.impactOccurred('light');
// При показе результата
tg.HapticFeedback.notificationOccurred('success'); // зелёная зона
tg.HapticFeedback.notificationOccurred('warning'); // жёлтая
tg.HapticFeedback.notificationOccurred('error');   // красная
```

### CloudStorage — сохранение прогресса без сервера

```js
// Сохранить текущий вопрос и ответы (до 4096 символов на ключ)
tg.CloudStorage.setItem('quiz_progress', JSON.stringify({ step: 3, answers: [2,1,3] }));
// Восстановить если закрыл и открыл снова
tg.CloudStorage.getItem('quiz_progress', (err, value) => { ... });
```

### Передача результата пользователю

**Рекомендуемый способ — через бэкенд:**
1. По завершении квиза — POST на бэкенд с `initData` (для верификации) + результаты
2. Бэкенд верифицирует `initData`, сохраняет результат, отправляет боту команду
3. Бот присылает пользователю в чат карточку с результатом + inline-кнопку CTA

Бот присылает в чат:
```
📊 Ваш результат диагностики

🔴 Критическая зависимость — 14 из 18

Три первых шага: ...

[Записаться на консультацию]
```

Это создаёт артефакт — пользователь может вернуться к нему, переслать партнёру, показать команде.

### Аналитика

Нативной аналитики нет. Варианты:
- **Яндекс.Метрика** — работает в WebView без ограничений, рекомендуется для РФ
- **Posthog / Mixpanel** — работают нормально
- `document.referrer` всегда пустой — источник трафика не определяется стандартными методами

### Ограничения TMA

- HTTPS обязателен
- Первый экран должен загружаться < 3 сек (Telegram показывает лоадер до `WebApp.ready()`)
- Рекомендуемый размер приложения < 2 МБ
- Нет доступа к камере, push-уведомлениям напрямую

## Экспертная оценка

---

### Взгляд UX-дизайнера

#### Типичные ошибки новичков (и как их избежать)

**1. Спрашивают то, что Telegram уже знает**
Форма с полями «Ваше имя» и «Телефон» — мгновенный дропаут. У вас этого нет — правильно. `first_name` берётся автоматически. Телефон в консалтинге не нужен — пишут в Telegram.

**2. Два CTA на экране результата**
«Записаться» + «Написать напрямую» = паралич выбора. Пользователь не делает ни того, ни другого. Оставить один путь. «Написать напрямую» убрать или спрятать как вторичную ссылку мелким текстом.

**3. Автопереход слишком быстрый или слишком медленный**
300 мс — минимум (нужно увидеть подсветку ответа). 600 мс — оптимум. 1000+ мс — кажется, что завис. Это мелочь, которую обычно не тестируют, и она режет completion rate.

**4. Экран загрузки без нарратива**
«Загрузка...» с крутящимся кружком — плохо. Три строки с ✓ (что уже сделано) и ⟳ (что считается прямо сейчас) — хорошо. У вас это описано правильно, не убирать под давлением «давай быстрее».

**5. Результат показывают цифрой, не типом**
«14 из 18 баллов» — ничего не говорит. «Критическая зависимость» — запоминается, хочется переслать, вызывает эмоцию. У вас это учтено. Убедитесь, что заголовок типа доминирует визуально над цифрой.

#### Что обязательно должно быть

| Элемент | Почему важен |
|---|---|
| Прогресс-бар с номером шага | Убирает тревогу «сколько ещё» — +20–28% completion |
| Подсветка выбранного ответа | Мгновенный фидбек = ощущение живого интерфейса |
| Экран загрузки 2–3 сек | Психологический триггер «настоящего анализа» |
| Один вопрос на экран | Мобильный формат, нет когнитивной нагрузки |
| BackButton на вопросах 2–6 | Пользователь должен иметь возможность исправить ответ |
| Персонализированный заголовок результата | Имя из Telegram («Ваш результат, Анна:») увеличивает вовлечённость |

#### Что бесит пользователей

- Кнопки слишком маленькие для пальца — минимум 48px высота, лучше 56–64px
- Текст мелкий или с низким контрастом — особенно в тёмной теме Telegram
- CTA появляется до того, как пользователь увидел ценность результата
- «Результаты отправлены» — и тишина. Пользователь не знает, что будет дальше
- Открытие внешнего браузера для записи — разрыв контекста, дропаут ~40%

---

### Взгляд маркетолога

#### Что заставит записаться

**1. «Зеркало» — точное попадание в ситуацию**
Если пользователь читает описание результата и думает «это написано про меня» — доверие к эксперту резко растёт. Каждое описание уровня должно содержать одну очень конкретную деталь (не «вы перегружены», а «команда ждёт вашего ответа, прежде чем сделать следующий шаг»). Проверить: дать прочитать 3–5 реальным предпринимателям из ЦА.

**2. Дефицит — один из сильнейших триггеров для B2B**
Добавить под кнопкой CTA: «Осталось 2 места на этой неделе» (и менять реально). Исследования по консалтингу показывают +15–25% к конверсии в заявку.

**3. Артефакт в чате — встроенный реферальный механизм**
Когда бот присылает красивую карточку результата в личный чат, её пересылают партнёрам и команде. Это органический трафик. Сделать карточку визуально привлекательной и добавить: «Пройдите диагностику: [ссылка]».

**4. Социальное доказательство в интро — снижает барьер входа**
«Уже прошли 847 предпринимателей» на приветственном экране. Начать с реального числа (первые 50, потом наращивать).

**5. Любопытство важнее пользы в заголовке**
«Узнайте, насколько бизнес зависит от вас» < «Проверьте: может ли ваш бизнес выжить без вас?». Второй вариант создаёт лёгкую тревогу и любопытство одновременно.

#### Элементы, повышающие конверсию

| Элемент | Эффект |
|---|---|
| Имя пользователя в результате | Персонализация повышает прочтение результата |
| Визуальный индикатор (красный круг) | Эмоциональный якорь — «я в красной зоне» = сильнее мотивация |
| Конкретные 3 шага (не 5 и не 1) | Даёт ощущение управляемости проблемы |
| Бесплатный звонок 20–30 мин (не час) | Снижает страх обязательств |
| Дефицит мест в CTA | +15–25% к заявкам по данным консалтинговых воронок |
| Карточка результата в боте | Виральность + возврат пользователя |
| «Разобрать вашу ситуацию» вместо «Консультация» | Глагол действия снижает барьер |

#### Что точно не работает для B2B-аудитории

- «Купить» или «Оплатить» — стоп-слова на первом касании
- Сбор email в Telegram-воронке — трение без пользы
- Длинный текст на экране результата (>6 строк без разбивки)
- Общие рекомендации типа «наладьте процессы» — вызывают недоверие к эксперту
- CTA на вебинар вместо личного звонка — конверсия ниже в 2–3x для горячего лида после квиза

### Взгляд Telegram-разработчика

#### Ограничения, которые нужно учитывать с самого начала

**1. `document.referrer` всегда пустой**
Источник трафика стандартными методами не определить. UTM-метки нужно передавать через `startParam` бота (`t.me/botname?start=utm_source_landing`) и парсить на бэкенде.

**2. `initDataUnsafe` без верификации — дыра в безопасности**
Всегда проверять на сервере через HMAC-SHA256 с бот-токеном. Клиентская проверка не считается. Если пропустить — любой может подделать userId и получить чужой результат.

**3. CloudStorage асинхронный и колбэчный**
Не promisify — работает медленно, могут быть задержки. Для 6 вопросов — сохранять прогресс в памяти и в CloudStorage параллельно, не ждать ответа хранилища перед переходом.

**4. Размер бандла < 2 МБ и загрузка < 3 сек**
Telegram показывает системный лоадер до вызова `WebApp.ready()`. Если > 3 сек — часть пользователей закрывает. Никакого React/Vue для 6 экранов — vanilla JS + минимум CSS, или Svelte в крайнем случае.

**5. Нет push-уведомлений из TMA**
Напоминание о записи можно отправить только через бота. Логика: пользователь прошёл квиз → не записался → бот пишет через 24 часа «Мы зарезервировали для вас время». Требует хранения состояния на бэкенде.

**6. Камера и микрофон недоступны**
Не планировалось, но важно знать на будущее — голосовые функции из TMA не реализуемы стандартными средствами.

#### Что работает отлично

**MainButton — самый конверсионный элемент TMA**
Нативная синяя кнопка внизу. Пользователи доверяют ей как кнопке Telegram, не как сайту. `showProgress()` на экране загрузки — встроенный спиннер, не нужен кастомный.

**HapticFeedback — делает квиз живым**
`impactOccurred('light')` на каждый выбор ответа. `notificationOccurred('success'/'warning'/'error')` на результат. Занимает 3 строки кода, эффект — пользователь чувствует приложение как нативное.

**themeParams — автоматическое следование теме**
CSS-переменные из `tg.themeParams` дают тёмную/светлую тему бесплатно. Не использовать хардкод цветов — в тёмной теме Telegram белый фон будет раздражать.

**Бот отправляет результат в чат — правильная архитектура**
Карточка результата в чате = постоянный артефакт. Пользователь возвращается к ней. Пересылает. Добавляет в избранное. Это работает лучше, чем страница «Ваш результат», которую нельзя найти через неделю.

#### Что лучше не делать

| Антипаттерн | Почему плохо |
|---|---|
| Кастомная кнопка «Назад» вместо BackButton | Нарушает нативное поведение, пользователи привыкли к BackButton |
| Кастомная кнопка внизу вместо MainButton | Теряется доверие «это часть Telegram», выглядит как сайт |
| Открывать запись в браузере | Разрыв контекста, дропаут ~40% |
| Тяжёлый фреймворк (Next.js, CRA) | Бандл > 500KB = загрузка > 3 сек = потеря трафика |
| Хранить userId без верификации initData | Уязвимость: подделка идентификатора |
| Запрашивать разрешения | Камера/микро недоступны, GPS — только через браузерный API, раздражает |
